### 3：S3のポリシーを変更する
前回構築したS3静的ウェブサイトのバケットポリシーが残ったままなので、このままでは`http://aws-lesson-2022.s3-website-ap-northeast-1.amazonaws.com`からでもアクセスが出来てしまう状況のため変更をしていく
#### 3.1 ブロックパブリックアクセスの変更
__【前回と今回の設定の変更】__
![スクリーンショット 2022-02-13 8.33.26.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/694365/d1c19585-3bf4-9f57-3795-9fc4b27d3282.png)

#### 3.2 バケットポリシーの変更
__【上記に伴うポリシーの変更】__
![スクリーンショット 2022-02-13 8.43.04.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/694365/afdf80d8-9392-81aa-f688-41e1a896f107.png)

```json:cloudfront-bucket-policy.json
{
	"Version": "2012-10-17",
	"Statement": [
			{
					"Effect": "Allow",
					"Principal": {
							"AWS": "arn:aws:iam::cloudfront:user/CloudFront Origin Access Identity ECOA39FPWOD7B"
					},
					"Action": "s3:GetObject",
					"Resource": "arn:aws:s3:::aws-lesson-20220212/*"
			}
	]
}
```
